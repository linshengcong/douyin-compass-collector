# 备份与恢复

## 备份范围

建议长期备份：

- `runtime/data/collector.db`；
- `runtime/exports/`。

按排障需要选备份：

- `runtime/raw/`；
- `runtime/artifacts/`；
- `runtime/logs/`。

默认不要备份或迁移 `runtime/browser-profile/`。其中包含登录凭证和浏览器本地状态，应在新 Mac 上重新执行 `login`。

## 备份前准备

先停止前台 Scheduler 或卸载 LaunchAgent，避免备份过程中继续写数据库：

```bash
./scripts/status_launchd.sh
./scripts/uninstall_launchd.sh
```

如果没有安装 LaunchAgent，只需确认终端中没有正在运行的 `scheduler`。

## SQLite 一致性备份

创建受限权限的备份目录：

```bash
mkdir -p runtime/backups
chmod 700 runtime/backups
```

使用 SQLite 在线备份命令，不直接复制正在使用的数据库文件：

```bash
sqlite3 runtime/data/collector.db ".backup 'runtime/backups/collector-backup.db'"
```

校验备份：

```bash
sqlite3 runtime/backups/collector-backup.db "PRAGMA integrity_check;"
```

结果必须为 `ok`。

CSV 可以在 Scheduler 停止后复制到受保护目录。不要把备份提交 Git 或上传到公开网盘。

## 恢复

1. 停止 Scheduler 或卸载 LaunchAgent；
2. 保存当前 `collector.db` 的额外副本；
3. 将通过完整性校验的备份恢复到 `runtime/data/collector.db`；
4. 执行 `uv run --frozen python -m compass_collector status`；
5. 运行一次 `--dry-run`；
6. 确认无误后再启动 Scheduler 或安装 LaunchAgent。

恢复后的数据库会在 CLI 启动时自动执行缺失的 Alembic 迁移。不要手工编辑 SQLite 表或 `alembic_version`。

## 新 Mac 迁移

只迁移仓库、SQLite 备份和必要 CSV。不要迁移旧 Chrome Profile；在新 Mac 上重新登录。LaunchAgent plist 包含旧机器绝对路径，也不要直接复制，应在新工程目录重新执行安装脚本生成。
